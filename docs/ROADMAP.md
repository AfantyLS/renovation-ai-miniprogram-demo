# 开发路线图

## 第 1 步：最小 Demo

状态：已完成基础骨架

目标：

- 用户可以进入小程序
- 用户可以填写装修需求
- 用户可以选择户型图
- 系统生成一份 mock 装修报告
- 用户可以查看报告

验收标准：

- 微信开发者工具可以正常编译
- 首页、创建页、报告页可以正常跳转
- 提交表单后可以看到报告内容

## 第 2 步：项目保存

目标：

- 增加本地项目列表
- 使用 `wx.setStorageSync` 保存历史方案
- 支持查看历史报告
- 支持删除 Demo 项目

建议文件：

- `pages/projects/`
- `utils/storage.js`

## 第 3 步：接入云开发

目标：

- 启用微信云开发
- 上传户型图到云存储
- 将用户需求保存到云数据库
- 使用云函数生成报告

建议云函数：

- `createProject`
- `generateReport`
- `getReport`

## 第 4 步：接入真实 AI

目标：

- 云函数或自建后端调用大模型
- 使用 JSON Schema 约束输出
- 生成结构化装修方案
- 失败时自动重试

注意：

- 不要在小程序前端放 API Key
- AI 输出必须校验后再入库
- 预算计算尽量由程序完成

## 第 5 步：效果图生成

目标：

- 每个主要房间生成效果图提示词
- 后端调用图片生成模型
- 图片保存到云存储
- 报告页展示图片

## 第 6 步：HTML 报告

目标：

- 后端生成 Web 版 HTML 报告
- 小程序内打开 H5 报告
- 支持分享链接
- 后续支持 PDF 导出

## 第 7 步：商业化增强

目标：

- 微信登录
- 用户项目管理
- 多版本方案对比
- 设计师人工优化
- 付费生成完整报告
- 品牌材料库和商品链接
